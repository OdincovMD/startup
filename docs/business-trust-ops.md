# Данные, безопасность, поддержка и юридический контур — Синтезум

Документ для B2B-заказчиков, DPO и операционки. **Не является юридической консультацией.** Публичные договоры и политики утверждаются с юристом.

Технический стек и развёртывание: [architecture.md](architecture.md), [development.md](development.md).

---

## 1. Данные и роли

| Категория | Примеры | Комментарий |
|-----------|---------|-------------|
| Учётные данные | Email, хэш пароля | `users` |
| Профиль человека | ФИО, фото, контакты, ORCID | Пользователь и профили соискателя/сотрудника |
| Контент организаций | Описания, запросы, вакансии | Публикуемый контент в каталогах |
| Поведенческие / аналитика | При наличии сбора событий | Уточнять по фактической конфигурации аналитики |

**Обработчик персональных данных** и **место хостинга** указываются в **Политике конфиденциальности** на публичном сайте (не дублировать здесь фиктивные реквизиты).

## 2. Безопасность (кратко для заказчика)

Реализация на уровне приложения (ориентиры из документации проекта):

- **Аутентификация:** JWT для API; подтверждение email; возможность инвалидации сессий (`token_version`); блокировка учётной записи администратором ([ENTITIES.md](ENTITIES.md)).
- **Авторизация:** роли (`student`, `researcher`, `lab_admin`, `lab_representative`, `platform_admin`); админ-API изолирован префиксом `/api/admin` ([admin-panel.md](admin-panel.md)).
- **Хранение паролей:** хэширование (не хранить в открытом виде).
- **Инфраструктура:** PostgreSQL, Elasticsearch, MinIO — доступ по сетевым политикам и секретам окружения ([development.md](development.md)).

**Что не входит в этот документ:** формальный ISO 27001, пентест-отчёты — предоставляются по запросу при наличии.

---

## 3. Резервное копирование и доступность

- Политика бэкапов БД и объектного хранилища задаётся **эксплуатацией** (частота, хранение, тест восстановления).
- **Целевой SLA аптайма** для публичного сервиса фиксируется в договоре с хостингом/подрядчиком, не в репозитории.

## 4. Поддержка и эскалации

Рекомендуемый внутренний регламент (заполнить контактами):

| Уровень | Примеры обращений | Канал | Целевое время первого ответа |
|---------|-------------------|-------|--------------------------------|
| L1 | Не могу войти, сброс пароля, вопрос по UI | Email / форма на сайте | Задать (например, 1 раб. день) |
| L2 | Спор о правах на организацию, жалоба на контент | Email + внутренняя эскалация | Задать |
| L3 | Инцидент безопасности, утечка данных | Выделенный контакт, вне почты | Немедленно |

**Инцидент:** недоступность сервиса для большинства пользователей, подозрение на компрометацию данных, массовая рассылка спама с платформы.

Администратор платформы действует по [admin-panel.md](admin-panel.md): блокировка пользователя, снятие публикации, удаление записей.

## 5. Outline для юриста: что вынести на сайт / в договор

Чеклист тем для подготовки документов (черновик требований к юристу):

1. **Пользовательское соглашение** — право использования сервиса, ограничение ответственности платформы за контент организаций, запрет злоупотреблений.
2. **Политика конфиденциальности** — цели обработки ПДн, категории данных, сроки, права субъекта, cookies (если есть), трансграничная передача (если применимо).
3. **Согласие на обработку ПДн** — если требуется отдельно от политики.
4. **Оферта / договор с юрлицом** — для платных подписок: предмет, срок, оплата, расторжение, возвраты (согласовать с [business-pricing.md](business-pricing.md)).
5. **Правила публикации** — краткая публичная версия раздела модерации из [business-journeys-faq.md](business-journeys-faq.md).
6. **Порядок рассмотрения жалоб** — на незаконный контент или персональные данные.

Юридические тексты **не хранить в репозитории** как единственный источник истины без версионирования с юристом; в репо допустим только outline (этот раздел) или ссылка на внешний документооборот.
