## 1. Ядро (пользователи, роли, подписки)

### 1.1 Роли (`roles`)

Роль пользователя в системе.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| name | Название роли (student, researcher, lab_admin, lab_representative, platform_admin) |

**Бизнес-назначение:** Определяет, какие действия доступны пользователю и какой профиль он заполняет.

---

### 1.2 Пользователи (`users`)

Учётная запись человека на платформе.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| public_id | Публичный идентификатор для URL профиля (например /applicants/abc123) |
| email | Email (логин) |
| hashed_password | Хэш пароля (если регистрация через email) |
| orcid | Идентификатор ORCID |
| openalex_id | Идентификатор в OpenAlex |
| role_id | Ссылка на роль |
| organization_id | Ссылка на организацию (для lab_admin, lab_representative) |
| full_name | ФИО |
| photo_url | URL фотографии |
| contacts | JSON: телефон, telegram и др. |
| email_verified | Подтверждён ли email |
| token_version | Версия токена (для инвалидации сессий) |
| created_at | Дата регистрации |

**Связи:**
- Один пользователь — одна роль
- Один пользователь — один профиль: Student ИЛИ Researcher ИЛИ Employee
- Пользователь может быть привязан к организации (для представителей)

**Бизнес-назначение:** Единая точка входа. В зависимости от роли пользователь видит разные разделы и возможности.

---

### 1.3 Подписки (`user_subscriptions`)

Подписка пользователя на тарифный план (Basic / Pro).

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| user_id | Пользователь |
| audience | Аудитория (representative — представители организаций) |
| tier | Тариф: basic или pro |
| status | Статус: active, cancelled |
| started_at | Дата начала |
| expires_at | Дата окончания |
| trial_ends_at | Дата окончания триала |
| activated_by | Кто активировал (admin) |
| cancelled_at | Дата отмены |
| discount_percent | Скидка в % |
| renewal_count | Количество продлений |

**Бизнес-назначение:** Ограничение возможностей для Basic (лимиты на вакансии, лаборатории, запросы) и снятие ограничений для Pro.

---

### 1.4 События подписок (`subscription_events`)

Аудит изменений подписки.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| subscription_id | Подписка |
| event_type | Тип события (created, extended, cancelled и т.п.) |
| performed_by | Кто выполнил действие |
| details | JSON с деталями |
| created_at | Время события |

## 2. Соискатели (студенты и исследователи)

### 2.1 Студенты (`students`)

Профиль соискателя-студента.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| user_id | Ссылка на пользователя |
| full_name | ФИО |
| status | Статус (например, «студент магистратуры») |
| skills | JSON: навыки |
| summary | Краткое резюме |
| resume_url | Ссылка на резюме |
| document_urls | JSON: ссылки на документы |
| education | JSON: образование |
| research_interests | JSON: научные интересы |
| is_published | Опубликован ли профиль |
| created_at | Дата создания |

**Бизнес-назначение:** Профиль для студентов, ищущих стажировки или работу. Виден в каталоге соискателей при is_published=true.

---

### 2.2 Исследователи (`researchers`)

Профиль соискателя-исследователя (кандидат наук, постдок и т.п.).

| Поле | Описание |
|------|----------|
| id | Идентификатор |

| organization_id | Текущая организация (опционально) |
| full_name | ФИО |
| position | Должность |
| academic_degree | Учёная степень |
| research_interests | JSON: научные интересы |
| education | JSON: образование |
| publications | JSON: публикации |
| hindex_wos, hindex_scopus, hindex_rsci, hindex_openalex | Индексы Хирша |
| job_search_status | Статус поиска работы |
| desired_positions | Желаемые должности |
| employment_type_preference | Предпочтение типа занятости |
| preferred_region | Предпочтительный регион |
| availability_date | Дата выхода на работу |
| salary_expectation | Ожидания по зарплате |
| job_search_notes | Заметки о поиске |
| resume_url | Ссылка на резюме |
| document_urls | JSON: документы |
| is_published | Опубликован ли профиль |
| created_at | Дата создания |

**Связи:** Исследователь может быть привязан к нескольким лабораториям (many-to-many через researcher_laboratories).

**Бизнес-назначение:** Профиль для учёных, ищущих позиции в лабораториях. Поддерживает заявки на присоединение к лаборатории.

## 3. Организации и лаборатории

### 3.1 Организации (`organizations`)

Организация (вуз, НИИ, компания).

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| public_id | Публичный идентификатор |
| name | Название |
| avatar_url | URL логотипа |
| description | Описание |
| address | Адрес |
| website | Сайт |
| ror_id | Идентификатор в ROR (Research Organization Registry) |
| is_published | Опубликована ли |
| creator_user_id | Кто создал |
| created_at, first_created_at | Даты создания |

**Связи:**
- Организация имеет лаборатории, оборудование, сотрудников, запросы, решения задач
- Организация может получать заявки на присоединение от лабораторий (org_join_requests)

**Бизнес-назначение:** Верхнеуровневая сущность. Лаборатории принадлежат организациям или работают автономно.

---

### 3.2 Лаборатории (`laboratories_organizations`)

Лаборатория — структурное подразделение, проводящее исследования.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| public_id | Публичный идентификатор |
| organization_id | Родительская организация (null = standalone) |
| creator_user_id | Кто создал |
| head_employee_id | Руководитель (сотрудник) |
| name | Название |
| description | Описание |
| activities | Направления деятельности |
| image_urls | JSON: фото |
| is_published | Опубликована ли |
| created_at, first_created_at | Даты создания |

**Связи:**
- Лаборатория принадлежит организации или автономна
- Имеет сотрудников (employees), исследователей (researchers), запросы, решения задач, оборудование
- Может подавать заявку на присоединение к организации (org_join_request)
- Исследователи могут подавать заявку на присоединение к лаборатории (lab_join_request)

**Бизнес-назначение:** Основная единица для поиска вакансий, запросов, оборудования. Может существовать без организации (standalone) с лимитами по тарифу Basic.

## 4. Вакансии и отклики

### 4.1 Вакансии (`vacancies_organizations`)

Вакансия организации или лаборатории.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| public_id | Публичный идентификатор |
| organization_id | Организация |
| laboratory_id | Лаборатория (если вакансия в конкретной лаборатории) |
| query_id | Связанный запрос (если вакансия по запросу) |
| creator_user_id | Кто создал |
| contact_employee_id | Контактное лицо (сотрудник) |
| contact_email, contact_phone | Прямые контакты |
| name | Название вакансии |
| requirements | Требования |
| description | Описание |
| employment_type | Тип занятости |
| is_published | Опубликована ли |
| created_at, first_created_at | Даты создания |

**Связи:**
- Вакансия принадлежит организации, может быть привязана к лаборатории и/или запросу
- Имеет отклики (vacancy_responses)

**Бизнес-назначение:** Позиция, на которую могут откликаться соискатели (студенты, исследователи).

---

### 4.2 Отклики на вакансии (`vacancy_responses`)

Отклик соискателя на вакансию.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| user_id | Соискатель (User) |
| vacancy_id | Вакансия |
| status | Статус: new, viewed, contacted, rejected |
| created_at, updated_at | Даты |

**Уникальность:** Один пользователь может откликнуться на одну вакансию только один раз.

**Бизнес-назначение:** Связь соискатель ↔ вакансия. Работодатель видит отклики и меняет статус.

## 5. Запросы и решения задач

### 5.1 Запросы организаций (`organization_queries`)

Запрос организации на решение задачи, поиск исполнителя или технологии.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| public_id | Публичный идентификатор |
| organization_id | Организация |
| creator_user_id | Кто создал |
| title | Заголовок |
| task_description | Описание задачи |
| completed_examples | Примеры выполненных работ |
| grant_info | Информация о гранте |
| budget | Бюджет |
| deadline | Сроки |
| status | Статус (active и др.) |
| linked_task_solution_id | Связанное решение задачи |
| is_published | Опубликован ли |
| created_at, first_created_at | Даты создания |

**Связи:**
- Запрос связан с лабораториями (many-to-many), сотрудниками (many-to-many)
- Может иметь вакансии
- Может быть связан с решением задачи (task_solution)

**Бизнес-назначение:** Запросы типа «ищем исполнителя», «решить задачу», «нужна технология». Исполнители и лаборатории могут откликаться.

---

### 5.2 Решения задач (`task_solutions_organizations`)

Описание решения задачи или проекта организации.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| organization_id | Организация |
| creator_user_id | Кто создал |
| title | Заголовок |
| task_description | Описание задачи |
| solution_description | Описание решения |
| article_links | JSON: ссылки на статьи |
| solution_deadline | Сроки |
| grant_info | Грант |
| cost | Стоимость |
| external_solutions | Внешние решения |
| created_at | Дата создания |

**Связи:**
- Решение связано с лабораториями (many-to-many)
- Запросы могут ссылаться на решение через linked_task_solution_id

**Бизнес-назначение:** Каталог решённых задач, проектов, кейсов. Может быть привязан к запросам.

## 7. Сотрудники

### 7.1 Сотрудники (`employees`)

Сотрудник организации (представитель, учёный, контакт).

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| organization_id | Организация |
| creator_user_id | Кто создал |
| user_id | Привязка к учётной записи (опционально) |
| full_name | ФИО |
| position | JSON: должности |
| academic_degree | Учёная степень |
| photo_url | Фото |
| research_interests | JSON: интересы |
| education | JSON: образование |
| publications | JSON: публикации |
| hindex_wos, hindex_scopus, hindex_rsci, hindex_openalex | Индексы Хирша |
| contacts | JSON: контакты |

**Связи:**
- Сотрудник принадлежит организации
- Сотрудник может быть привязан к пользователю (user_id) — тогда он входит в систему
- Сотрудник может быть в нескольких лабораториях (many-to-many)
- Сотрудник может быть привязан к запросам (many-to-many)
- Сотрудник может быть руководителем лаборатории (head_employee)
- Сотрудник может быть контактным лицом по вакансии

**Бизнес-назначение:** Персонал организаций. Может иметь аккаунт (user_id) или быть только карточкой. Используется в лабораториях, вакансиях, запросах.

## 8. Заявки на присоединение

### 8.1 Заявки исследователя в лабораторию (`lab_join_requests`)

Заявка исследователя на присоединение к лаборатории.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| researcher_id | Исследователь |
| laboratory_id | Лаборатория |
| status | Статус: pending, approved, rejected |
| created_at, updated_at | Даты |

**Уникальность:** Один исследователь — одна заявка на одну лабораторию.

**Бизнес-назначение:** Исследователь подаёт заявку, представитель лаборатории принимает или отклоняет. При одобрении исследователь привязывается к лаборатории.

---

### 8.2 Заявки лаборатории в организацию (`org_join_requests`)

Заявка лаборатории на присоединение к организации.

| Поле | Описание |
|------|----------|
| id | Идентификатор |
| laboratory_id | Лаборатория |
| organization_id | Организация |
| status | Статус: pending, approved, rejected |
| created_at, updated_at | Даты |

**Уникальность:** Одна пара лаборатория–организация.

**Бизнес-назначение:** Standalone-лаборатория хочет войти в организацию. Представитель организации одобряет или отклоняет.

## Схема ролей и профилей

```
User
 ├─ role: Role (student | researcher | lab_admin | lab_representative | platform_admin)
 ├─ student_profile: Student?      (если роль student)
 ├─ researcher_profile: Researcher? (если роль researcher)
 └─ employee_profile: Employee?     (если роль lab_admin или lab_representative)

Organization
 ├─ laboratories
 ├─ employees
 ├─ equipment
 ├─ queries
 ├─ task_solutions
 └─ vacancies

OrganizationLaboratory
 ├─ organization
 ├─ employees
 ├─ researchers (через researcher_laboratories)
 ├─ equipment
 ├─ queries
 └─ task_solutions
```
