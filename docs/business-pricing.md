# Тарифы и коммерческая упаковка — Синтезум

Документ для закупок, финансового директора и продаж: **что покупает клиент** простым языком. Техническая и алгоритмическая точность — в [subscription-ranking.md](subscription-ranking.md); сущности — в [ENTITIES.md](ENTITIES.md).

---

## 1. Что продаётся

**Продукт:** доступ представителя организации/лаборатории к расширенным возможностям платформы Синтезум в виде **подписки на пользователя** (учётная запись с ролью представителя).

Подписка **не привязана к организации как к юрлицу** в первой фазе: оформляется на **пользователя** (`user_subscriptions.user_id`). Распространение преимуществ на лаборатории организации при тарифе Pro описано ниже и детально в [subscription-ranking.md](subscription-ranking.md) (Org-wide).

---

## 2. Тарифы (язык закупки)

### 2.1. Basic

**Обещание для договора/КП:** «Стартовый коммерческий доступ: снятие жёстких лимитов на масштаб по сравнению с бесплатным использованием в части количества размещений; участие в **платном блоке** каталогов наравне с Pro по правилам ранжирования».

**Лимиты (контроль злоупотреблений):**

| Возможность | Basic |
|-------------|--------|
| Организации | 1 **или** самостоятельные лаборатории без орг. |
| Самостоятельные лаборатории (без привязки к организации) | до **3** |
| Лаборатории внутри созданной организации | без отдельного лимита по числу |
| Вакансии (совокупно по создателю) | до **15** |
| Запросы (совокупно по создателю) | до **15** |

Источник правил: [subscription-ranking.md](subscription-ranking.md), раздел «Ограничения Basic».

### 2.2. Pro

**Обещание для договора/КП:** «Полный масштаб без лимитов Basic на вакансии и запросы; **организационный** эффект: лаборатории, вступившие в организацию по правилам платформы, могут **наследовать платный статус** от организации при активной подписке Pro у создателя организации (Org-wide)».

**Лимиты:** нет потолков Basic по перечисленным сущностям.

---

## 3. Что получает клиент помимо лимитов

- **Видимость:** платные сущности попадают в **верхний блок** выдачи каталогов (до разделения на бесплатный блок); при текстовом поиске действует связка релевантности и `rank_score` (см. [subscription-ranking.md](subscription-ranking.md)).
- **Пробный период (trial):** может быть задан при активации (`trial_ends_at`) — в договоре фиксировать длительность и условие перехода на платный период.
- **Grace period:** первые **7 дней** после создания первой организации/лаборатории — boost в выдаче без подписки (маркетинговый стимул онбординга; см. спецификацию).

## 4. Процесс оплаты и биллинга (текущая фаза продукта)

**Как задокументировано в продукте сегодня:**

- Активация подписки — **через администратора платформы** (ручная выдача), не через автоматический платёжный шлюз в первой фазе ([subscription-ranking.md](subscription-ranking.md)).
- Пользователь может подать **заявку на подписку** (`subscription_requests`); администратор одобряет или отклоняет ([ENTITIES.md](ENTITIES.md), [admin-panel.md](admin-panel.md)).

**Что вынести во внешний договор/оферту (с юристом):**

- Предмет: право использования функций тарифа в течение периода.
- Срок, продление, расторжение, заморозка (если появится в продукте).
- Последствия **истечения** подписки: возврат к лимитам Basic/бесплатного режима и потеря платного блока в выдаче — описать пользовательски.
- Персональные данные при оплате (если позже подключается эквайринг) — см. [business-trust-ops.md](business-trust-ops.md).

## 5. ROI и аргументация для заказчика

Тезисы для переговоров; **цифры без исследований не придумывать** — помечать как гипотезу.

| Тезис | Комментарий |
|-------|-------------|
| Сокращение времени на «поиск лаборатории» за счёт структурированного каталога и запросов | Логическая ценность; измерять через интервью NPS |
| Рост охвата соискателей по вакансиям в научной нише vs только локальный сайт | **Гипотеза** до A/B или статистики |
| Прозрачность компетенций (оборудование, сотрудники, описание) снижает число «холостых» контактов | Качественная гипотеза |
| Платное размещение как аналог медийной закупки с предсказуемым местом в выдаче | Согласовать с фактическим UI |
| Org-wide Pro для кампуса с множеством лабораторий — единая подписка на видимость юнитов | Сильный аргумент для НОЦ/вуза **при** согласованной цене |

## 6. Скидки и лояльность

В модели предусмотрены поля `discount_percent`, `renewal_count`, аудит `subscription_events` ([subscription-ranking.md](subscription-ranking.md)). Коммерческая политика скидок (первый год, вуз-партнёр) — вне кода; здесь только напоминание синхронизировать **публичные обещания** с тем, что админ вносит в систему.
